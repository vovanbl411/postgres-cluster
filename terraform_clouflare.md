# Развертывание защищенного Bastion-хоста через Cloudflare Zero Trust и Terraform в Timeweb Cloud

## 1. Архитектурная цель

Задача: Обеспечить безопасный SSH-доступ к изолированному кластеру баз данных (PostgreSQL) в приватной сети (VPC) Timeweb Cloud.
Ограничения: * Отказ от публикации публичных IP-адресов для бастиона (снижение поверхности атаки).

 * Автоматизация развертывания через GitHub Actions (CI/CD) с использованием Terraform.
 * Использование Cloudflare Tunnel для проброса трафика.

## 2. Стек технологий

 * Провайдер: Timeweb Cloud (вычислительные ресурсы, VPC).
 * Сеть: Cloudflare Zero Trust (Argo Tunnel, DNS).
 * IaC: Terraform (провайдеры twc и cloudflare).
 * ОС Бастиона: Debian 13 (кастомный образ с предустановленными утилитами).

## 3. С чем мы столкнулись (Проблемы и Грабли)

В процессе автоматизации деплоя мы поймали несколько критических ошибок, блокирующих пайплайн:

#### Проблема: Ошибка Authentication error (10000) в провайдере Cloudflare

 * Симптомы: Terraform падал при попытке создать ресурс cloudflare_zero_trust_tunnel_cloudflared.ssh_tunnel.
 * Причина: В Terraform передавался API-токен, имеющий права только на редактирование DNS-зоны (Zone.DNS). Создание туннелей требует прав уровня аккаунта. Попытка использовать раздельные токены (один для туннеля, другой для DNS) в одном блоке provider "cloudflare" не работает.
 * Решение: Был создан единый "Token" в панели Cloudflare, объединяющий необходимые права:
   * Account -> Cloudflare Tunnel -> Edit
   * Zone -> DNS -> Edit



## 4. Итоговое решение (Как это работает сейчас)

 1. GitHub Actions запускает Terraform и передает ему учетные данные (Timeweb Token, Cloudflare API Token, R2 Backend ключи) через Secrets.
 2. Terraform генерирует случайный 64-символьный пароль (random_password) для туннеля.
 3. В Cloudflare создается сущность туннеля и генерируется токен подключения (tunnel_token).
 4. Настраивается маршрутизация Cloudflare (ingress_rule), перенаправляющая трафик с внешнего домена (например, bastion.domain.com) на ssh://localhost:22 внутри бастиона.
 5. Создается CNAME-запись в DNS зоне Cloudflare, указывающая на UUID туннеля (<UUID>.cfargotunnel.com).
 6. В Timeweb Cloud поднимается сервер из заранее подготовленного кастомного образа (image_id).
 7. Через механизм user_data на старте сервера утилите cloudflared скармливается tunnel_token, полученный из Terraform, после чего туннель переходит в статус HEALTHY.

## 5. Доступ к инфраструктуре

Для подключения к узлам кластера через бастион используется проксирование локального SSH-клиента:
Настройка ~/.ssh/config локальной машины:
Host bastion.*
    ProxyCommand /usr/bin/cloudflared access ssh --hostname %h
    User root
    IdentityFile ~/.ssh/ansible_key

Команда для входа: ssh bastion.your-domain.com
*После входа на бастион открывается доступ к L3-маршрутизации приватной сети (VPC) 192.168.10.0/24 для управления Postgres-нодами.*
Как тебе такой формат? По-моему, выглядит солидно и покрывает все наши приключения с токенами и терраформом. Если хочешь, можем добавить сюда раздел про то, как собирали сам кастомный образ бастиона в Timeweb.
